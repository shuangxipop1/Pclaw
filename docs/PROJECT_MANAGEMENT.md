# Pclaw 项目管理系统

## 一、项目组织架构

### 1.1 项目管理层级

```
项目经理 (Project Manager)
    │
    ├── 设计经理 (Design Manager)
    │       └── 各专业工程师
    │
    ├── 采购经理 (Procurement Manager)
    │       └── 各专业工程师
    │
    ├── 施工经理 (Construction Manager)
    │       └── 各专业工程师
    │
    ├── 控制经理 (Control Manager)
    │       └── 进度/费用控制工程师
    │
    ├── 费控经理 (Cost Control Manager)
    │       └── 费控工程师
    │
    ├── 财务经理 (Finance Manager)
    │       └── 财务专员
    │
    ├── 文控经理 (Document Control Manager)
    │       └── 文档工程师
    │
    └── 商务经理 (Business Manager)
            └── 商务专员
```

### 1.2 职责说明

| 角色 | 职责 |
|------|------|
| 项目经理 | 全面负责项目对接、统筹协调、决策审批 |
| 设计经理 | 设计进度/质量管控、设计团队管理 |
| 采购经理 | 供应商管理、物资采购、合同执行 |
| 施工经理 | 现场管理、施工进度、安全质量 |
| 控制经理 | 进度管理、变更管理、风险控制 |
| 费控经理 | 费用预算、成本分析、变更核算 |
| 财务经理 | 资金管理、付款审核、财务报表 |
| 文控经理 | 文档收发、档案管理、版本控制 |
| 商务经理 | 市场对接、招投标、合同谈判 |

## 二、项目文件夹体系

### 2.1 管理类文件夹

| 文件夹 | 说明 |
|--------|------|
| 项目进度管理 | 项目计划、进度跟踪、里程碑 |
| 变更管理 | 变更申请、审批流程、变更记录 |
| 文档管理 | 来往函件、审批文件、会议纪要 |
| 质量管理 | 质量计划、检查报告、不符合项 |
| 安全管理 | 安全预案、检查记录、事故报告 |

### 2.2 技术类文件夹

| 文件夹 | 说明 |
|--------|------|
| 设计规范 | 设计标准、设计输入/输出 |
| 船级社 | ABS/DNV/CCS 规范要求 |
| 标准规范 | GB/ISO/API 标准库 |
| 行业最佳实践经验 | 案例库、经验总结 |
| 行业软件资源 | 软件清单、授权信息 |

### 2.3 商务类文件夹

| 文件夹 | 说明 |
|--------|------|
| 供应商寻源 | 供应商名录、评价资料 |
| 历史合同 | 过往合同、报价单 |
| 经验教训 | 项目复盘、教训总结 |
| 项目招投标信息 | 招标文件、投标文件 |
| 行业新闻 | 市场动态、政策法规 |
| 产品样本 | 产品介绍、技术参数、图片视频 |
| 产品业绩 | 过往项目成果、案例展示 |
| 客户资料 | 客户信息、联系人、沟通记录 |
| 商机 | 潜在项目机会、跟进记录 |
| 重点跟踪项目 | 高价值项目、优先推进 |

## 三、功能特性

### 3.1 角色权限管理
- 基于角色的访问控制 (RBAC)
- 灵活的权限配置
- 操作日志审计

### 3.2 文档管理
- 版本控制
- 在线预览
- 权限共享
- 全文检索

### 3.3 进度管理
- WBS 分解
- 关键路径分析
- 进度预警
- 资源加载

### 3.4 变更管理
- 变更申请流程
- 影响分析
- 审批流转
- 变更追踪

### 3.5 费用控制
- 预算分解
- 实际成本跟踪
- 偏差分析
- 预测调整

---

## 四、使用场景

- **工程建设**：船舶、海洋工程、陆地设施
- **设计咨询**：工程设计、项目管理
- **装备制造**：设备采购、建造监造
- **EPC 总包**：设计采购施工一体化
