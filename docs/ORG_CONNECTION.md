# Pclaw 组织连接功能设计

## 设计理念

每个运行 Pclaw 的机器背后都有一个人，这个人是为 Pclaw 的产出负责的，是属于真实架构中的一份子，需受组织管理的约束与管理。

---

## 一、核心功能

### 1.1 上下级连接

- 每个 Pclaw 可以连接多个上级 Pclaw
- 每个 Pclaw 可以连接多个下级 Pclaw
- 连接关系：树形结构，可有多级

### 1.2 汇报文件夹

- 上级 Pclaw 具备查询下级 Pclaw 工作专属汇报文件夹的权利
- 有多个上级时，分别建立多个汇报文件夹
- 每个 Pclaw 对自己的工作汇报文件夹负责

### 1.3 审批机制

- 放入文件夹的内容需经用户批准
- 审批流程：创建 → 用户审批 → 放入正式文件夹

---

## 二、数据结构

### 2.1 组织关系配置

```json
{
  "org": {
    "pclawId": "唯一标识",
    "role": "项目经理/设计经理/...",
    "parent": ["上级Pclaw ID列表"],
    "children": ["下级Pclaw ID列表"],
    "reportFolders": {
      "上级ID1": "/path/to/report/folder1",
      "上级ID2": "/path/to/report/folder2"
    }
  }
}
```

### 2.2 汇报文件

```
汇报文件夹/
├── 待审批/
│   └── 2026-03-12-项目进展.md
├── 已通过/
│   └── 2026-03-11-设计成果.md
└── 已驳回/
    └── 2026-03-10-预算申请.md
```

---

## 三、功能模块

### 3.1 组织连接管理
- 添加/删除上下级连接
- 验证连接有效性
- 同步组织架构

### 3.2 汇报文件夹
- 创建汇报文件夹
- 查看下级汇报
- 文件夹权限管理

### 3.3 审批流程
- 提交汇报内容
- 用户审批/驳回
- 自动归档

### 3.4 通知机制
- 新汇报提醒
- 审批结果通知
- 下级状态监控
